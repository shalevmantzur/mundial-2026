## 🚀 העלאה לרשת — הדרך הנוחה (2 דקות)

הפרויקט הוא תיקיית קבצים סטטית. הדרך הכי קלה: **גרירה לאתר Netlify Drop**.

1. כנס ל-https://app.netlify.com/drop
2. גרור את **כל תיקיית הפרויקט** (או את קובץ ה-ZIP) לתוך החלון
3. תוך שניות תקבל כתובת חיה כמו `https://random-name.netlify.app` — שתף אותה עם חברים

זהו. האתר באוויר. (אפשר גם Vercel, Cloudflare Pages, או GitHub Pages — כל שירות אחסון סטטי חינמי עובד.)

> **₪40 שנשארו בכיס:** התשתית חינמית לגמרי. אם תרצה דומיין יפה משלך (`mundial-shelanu.com`) — זה המקום היחיד שעולה כסף, וזה אופציונלי.

## 🟢 מעבר למצב חי (ריבוי משתמשים אמיתי)

כדי שחברים יוכלו להירשם מהמכשירים שלהם ולראות טבלה משותפת, צריך בסיס נתונים. נשתמש ב-**Supabase** (חינמי עד גדלים גדולים בהרבה מ-50 משתמשים).

### שלב 1 — יצירת פרויקט Supabase
1. כנס ל-https://supabase.com והירשם (חינם)
2. לחץ **New Project**, תן שם וסיסמה למסד הנתונים, בחר אזור קרוב (Frankfurt)
3. המתן ~2 דקות עד שהפרויקט מוכן

### שלב 2 — הרצת הסכמה
1. בתפריט הצד בחר **SQL Editor** → **New query**
2. פתח את הקובץ `schema.sql`, העתק את כל התוכן, הדבק, ולחץ **Run**
3. אמור להופיע "Success" — נוצרו הטבלאות, ההרשאות ומשחקי הדגמה

### שלב 3 — חיבור האפליקציה
1. ב-Supabase: **Project Settings** (גלגל שיניים) → **API**
2. העתק שני ערכים:
   - **Project URL** (לדוגמה `https://abcd1234.supabase.co`)
   - **anon public** key (מפתח ארוך)
3. פתח את `config.js` והדבק אותם:
   ```js
   window.MUNDIAL_CONFIG = {
     SUPABASE_URL: "https://abcd1234.supabase.co",
     SUPABASE_ANON_KEY: "eyJhbGci...",
     ADMIN_USERNAME: "השם-שלך"   // ← שנה לשם המשתמש שלך
   };
   ```
4. שמור, והעלה מחדש את הקבצים (גרירה ל-Netlify שוב).

### שלב 4 — הפיכתך למנהל
שם המשתמש שכתבת ב-`ADMIN_USERNAME` יהפוך אוטומטית למנהל ברגע שתירשם איתו באפליקציה. שאר המשתמשים יהיו רגילים.

> **הערה על אבטחה:** מפתח ה-`anon` נועד להיחשף בצד הלקוח — זה תקין. ההגנה על הנתונים נעשית דרך כללי ה-RLS שכבר מוגדרים ב-`schema.sql` (כל אחד עורך רק את הניחושים שלו; רק מנהל מזין תוצאות).

## ⚙️ אזור הניהול

נגיש רק למנהל (טאב **ניהול** בתפריט התחתון). שני חלקים:

**💰 ניקוד** — קבע כמה נקודות שווה כל סוג ניחוש (תוצאה מדויקת, מנצחת, כובש ראשון, כובש, מבשל, שתי הקבוצות כובשות, נבחרת אלופה, מלך שערים, מלך בישולים). השינוי נשמר מיד ומשפיע על החישובים.

**📥 תוצאות** — לכל משחק: הזן תוצאה סופית, רשימת כובשים (הראשון ברשימה = הכובש הראשון), ורשימת מבשלים. בלחיצה על "שמירה וחישוב נקודות" המערכת מנקדת אוטומטית את כל הניחושים של כל המשתמשים לפי החוקים שקבעת.

## 🐞 פאנל הדיבאג

כפתור צף (🐞) בפינה התחתונה — זמין בכל מסך. נבנה כדי שתוכל לאבחן ולתקן בעיות בעצמך:

- **מצב וחיבור** — מראה אם אתה בדמו או חי, והאם Supabase מחובר
- **מונה שגיאות ואזהרות** — אם יש שגיאה, הכפתור מהבהב באדום
- **יומן חי** — כל פעולה (הרשמה, שמירת ניחוש, חישוב נקודות, קריאות שרת) מתועדת עם זמן מדויק
- **🔌 בדיקת חיבור** — בודק שה-Supabase מגיב, ומציג זמן תגובה
- **📋 העתק דוח** — מעתיק דוח מלא (מצב, דפדפן, כל הלוגים) — שימושי אם תרצה שאעזור לאבחן תקלה: פשוט הדבק לי אותו
- **🗑️ נקה** / **♻️ איפוס דמו** — ניקוי לוגים או איפוס נתוני הדמו המקומיים
- **סינון** — הצג רק שגיאות / אזהרות / מידע

**אם משהו לא עובד:** פתח את הפאנל, לחץ "העתק דוח", ושלח לי את הטקסט. רוב הבעיות (מפתח שגוי, סכמה שלא רצה) יהיו ברורות מיד מהלוג.

## 🔄 עדכון נתונים אוטומטי (אופציונלי, חינמי)

כדי לחסוך הזנה ידנית של תוצאות, אפשר לחבר API חינמי. שתי אפשרויות נבדקו:

1. **openfootball/worldcup.json** — נתונים בנחלת הכלל, ללא מפתח, עדכון קל באיחור אחרי כל משחק. הכי יציב.
   `https://github.com/openfootball/worldcup.json`
2. **rezarahiminia/worldcup2026** — REST API חינמי עם עדכון בזמן אמת במהלך הטורניר.
   `https://github.com/rezarahiminia/worldcup2026`

**הגישה המומלצת (היברידית):** ה-API מזין אוטומטית את רוב הנתונים (תוצאות, שערים, מבשלים), ואזור הניהול נשאר כגיבוי לתיקון או השלמה ידנית של מה שה-API פספס. כך מקבלים אוטומציה כמעט מלאה עם רשת ביטחון — והכל בחינם. (החיבור עצמו דורש מעט קוד נוסף ב-`app.js`; אשמח להוסיף אותו כשתרצה.)
